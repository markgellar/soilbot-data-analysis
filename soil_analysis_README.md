# Soil Analysis — Lancaster County, PA

Python tools for processing USDA SSURGO soil survey data and generating spatially interpolated soil property maps using Kriging. Built to support autonomous soil sampling research with the [SoilBot simulation](https://github.com/markgellar/soilbot-sim).

## What This Does

### SSURGO Data Processing

Reads raw SSURGO tabular data (mapunit, component, and horizon tables) for Lancaster County, PA (survey area PA071). Filters to the top 10 cm of soil (probe depth), joins the three tables together, and enriches with sand/silt/clay texture data from the USDA Soil Data Access API. Provides a `soil_lookup(lat, lon)` function that takes a GPS coordinate and returns the soil properties for that location by matching against SSURGO polygon shapefiles.

Properties extracted per location include pH, organic matter, cation exchange capacity (CEC), sand/silt/clay percentages, available water capacity, saturated hydraulic conductivity, and soil classification.

### Kriging Maps

Uses Ordinary Kriging to build spatially interpolated maps of key soil properties across the county. Generates prediction and variance maps for CEC, sand %, silt %, clay %, and organic matter %, masked to the county boundary. Variance maps show where the interpolation is least certain, which can inform where to prioritize physical sampling.

## Data Source

All soil data comes from the [USDA NRCS SSURGO database](https://www.nrcs.usda.gov/resources/data-and-reports/soil-survey-geographic-database-ssurgo) for survey area **PA071 (Lancaster County, PA)**. Texture data is fetched from the [Soil Data Access (SDA) API](https://sdmdataaccess.sc.egov.usda.gov/) on first run and cached locally.

You will need the SSURGO download for PA071 placed in a `PA071/` folder alongside the scripts, containing the `tabular/` and `spatial/` subdirectories and the `soildb_PA_2003.mdb` file.

## Usage

### Process SSURGO data

```bash
python ssurgo_processing.py
```

Outputs `SSURGO_results.csv` with all joined soil properties. On first run, fetches texture data from the SDA API and caches it to `texture_lookup.csv` for offline use.

### Generate Kriging maps

```bash
python kriging_maps.py
```

Outputs `kriging_all_variables.png` — a 2x5 grid showing prediction and variance maps for each soil property, masked to the Lancaster County boundary.

### Look up soil at a coordinate

```python
from ssurgo_processing import soil_lookup, soilmap, merged

result = soil_lookup(39.870045, -76.185195, soilmap, merged)
print(result["ph1to1h2o_r"])  # pH
print(result["om_r"])          # organic matter %
print(result["cec7_r"])        # CEC meq/100g
```

```
    ├── tabular/
    │   ├── mapunit.txt
    │   ├── comp.txt
    │   └── chorizon.txt
    ├── spatial/
    │   └── soilmu_a_pa071.shp (+ .dbf, .shx, .prj)
    └── soildb_PA_2003.mdb
```
